Not sure about the other issues you reported. I haven't seen those before.

However, note that latest X2Go Server packages (3.1.1.4) know about  
GNOME and Unity and it should be possible to select a GNOME session as  
desktop session in X2Go Client. For Unity you can select Unity. The  
server-side X2Go tools will detect the OS environment of the server  
and launch the requested desktop session (GNOME or Unity, depends on  
your selection in X2Go Client).
