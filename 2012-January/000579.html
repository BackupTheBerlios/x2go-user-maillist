We added Xinerama support to x2goagent/nxagent. It feels as if this  
fails with xfreerdp.

With the latest version of x2goclient (nightly builds) you can  
enable/disable the xinerama feature in the session profile (near the  
other display settings like full screen, dpi, etc.). You also have to  
install latest NX (aka x2goagent) on the server for this to work.  
Could you check if disabling xinerama support in the session profile  
will ship around your problem? If xinerama is already disabled in your  
session profile, please test nonetheless. Before the option was  
introduced (yesterday), it was enabled by default and not shown in the  
session profile manager.
