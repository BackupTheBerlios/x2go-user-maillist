You setup is possible but not trivial. Normally clients must have  
access to the X2Go Servers. You can handle that by SSH-proxying  
through the broker machine. You do not need the broker for this to be  
set up, you only need the SSH daemon on the broker machine for this.

So my recommendation is that you test this with X2Go Client in normal  
session config mode first. If that works, you transfer the sshproxy*  
options in ~/.x2goclient/sessions to the broker configuration file  
(x2gobroker-sessionprofiles.conf).
