Hi Barnet,

On Fr 14 Jun 2013 15:43:23 CEST Barnet Wagman wrote:

> I need to know how connections failure are handled by x2go.
>
> E.g. Suppose a session is running a gui application (such as  
> Netbeans) in gnome on the server, and the connection is lost.  Is  
> the application killed?  Or is it possible to reconnect to the same  
> session?
>
> thanks

If the connection is lost, then the server-side GNOME session (X2Go)  
will be suspended and can be resumed at a later point of time.  
Processes inside the session are not frozen, but remain intact and  
running (like with the screen command for shells).
