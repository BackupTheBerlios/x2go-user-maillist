> I can confirm that 3.0.99 on W7 x64 won't connect. I have server

> Only 3.01-13 works on W7 x64. If I interpreted the changelogs

I've finally had success connecting using a client newer than 3.01-13 on W7 x64.

I installed a clean W7 x64 in a virtual machine and 3.99.0.0 worked.
After that I uninstalled x2goclient and pyhoca-gui from my primary W7
x64 client, manually cleaned all traces of x2go/ssh/vcxsrv from the
registry and user profile, deinstalled Symantec Endpoint Protection,
disabled Windows Firewall and installed 3.99.0.0 ... and it works!

Not sure what's the root cause of the problems, I'm a little
suspicious of Symantec Endpoint Protection. Will do some more testing
to see if I can figure it out.
