The user has to make sure that he can SSH to the machine with the LDAP  
user accounts. The login must produce a home directory to the user  
(probably via NFS?). If an SSH login works as expected, then X2Go  
should not be a problem.

Note: in X2Go Client there are some LDAP settings in some  
configuration dialog. This is misleading. The LDAP support in X2Go  
Client is obsolete and will soon be fully replaced by the X2Go Session  
Broker functionality. The LDAP feature in X2Go Client has __nothing__  
to do with the actual login to the remote server.

Make sure that SSH works and then X2Go works.
