Hi Aron,

On Mo 29 Nov 2010 13:02:58 CET Aron wrote:

> When I mount a shared folder with X2go client for Windows, the new files
> created doesn't correspond to the SSH user.
> Instead, they get this owner user and group: 1003:513
>
> Thanks
>

you can ignore this. The windows-side ssh daemon shows us the user's  
UID number and GID number. These are normally not seen on windows  
machines unless you play with CIFS/SMB and a Samba server under Linux.  
(There are probably other ways.)

However, the shared folder on the X2go server side is an sshfs mount  
point that does normally does not / cannnot map user names und UID  
numbers that exist on the client. Thus, you see UID and GID numbers  
that are unknown to the X2go server. It can be even worse, as a UID  
number of 1003 can be mapped to a server-side username that has the  
server-side UID number 1003. Funnily this 1003-user will not have  
access to sshfs share...

You can make your own tests, of course, but basically the only user  
who has access to an SSHFS share is the user who mounted the share via  
fuse in the first place, not matter what uids and gids on the remote  
share show in ,,ls -al'' command output.
