Hi folks,
Today I can answer myself. The problem is related with the keyboard type (e.g. 
pc-101,pc-105 etc.). When suspend, then resume session with different keyboard 
type, key mapping gets wrong and all other kbd related things are in trouble. 
Maybe this can be addressed somehow in the server/client code so it produce a 
warning or force setting the 'right' kbd type. This will be a problem for 
roaming users resuming their sessions from various computers.
