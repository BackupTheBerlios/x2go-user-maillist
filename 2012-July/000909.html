Dear all,

The X2Go Project is proud to announce a couple of new X2Go component releases:

   - python-x2go (0.2.0.6)
   - pyhoca-gui (0.2.0.1)
   - x2gognomebindings (2.0.1.6)

All new packages will be available in our stable repositories (Debian,  
     Ubuntu) within the next 24h.

The Windows version of PyHoca-GUI will be available within the next 48 hours.

The new PyHoca-GUI and Python X2Go versions are mainly bugfixing releases.

The new X2Go Gnome(2) Bindings package fixes a problem with unmounting  
of client-side shares if the user's HOME is mounted via NFS. Note:  
this package is only installable on GNOME 2 installations (Ubuntu  
Lucid, Debian squeeze). We are thinking about how to upgrade this  
package for usage with GNOME 3.

light+love,
Mike

   * New upstream version (0.2.0.6)
     - Ignore non-master sessions before calling the foldersharing-
       not-available hook.
     - Add several double underscore method aliases in X2goSession class.
     - Add several double underscore method aliases in X2goClient class.
     - Avoid double notifications on SSHFS being unavailable for the
       authenticated user.
     - X2goSession.HOOK_printing_not_available(): Fix log message.
