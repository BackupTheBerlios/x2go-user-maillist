As far as I can see, the x2go client keyboard settings are not respected 
when starting a KDE session on the server because KDE overrides these 
with its own settings.

For our NX server server I was able to override the KDE settings by 
setting UserScriptAfterSessionStart in node.cfg to reference a script 
that parsed the NX session keyboard file and constructed the appropriate 
setxkbmap command.

Is there an equivalent to the UserScriptAfterSessionStart functionality 
for the x2go server?
