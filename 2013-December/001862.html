> We are running a forex trading program inside of wine on an Edubuntu 12.04
> LTSP server. We have 5 test users setup on the server each running their
> own trading session, each with their own install of the program in wine.
> The trading program is left running for each user.
>
> I can login to any user on the server with x2go but the trading program is
> not available when I do, even though it is running for that same user on
> the server. If I try to start the software from the x2go session it briefly
> flickers ;it it tries to open but quickly dies. Is there any way to "tap"
> into that same session running on the server once I am logged in? I guess
> what I am really asking is whether X2go can run just to enable "viewing of

This is only possible if the running session is a desktop session.  
Then you can try to use the X2Go Client session type: Connection to  
local desktop.
