I was unable to reproduce your bug on this setup:
- Fedora 19 64-bit with XEmacs 21.5.34, x2goserver 4.0.1.6 & nx-libs 3.5.0.21
- Windows 8.1 64-bit with x2goclient 4.0.1.2pre02 (see the X2go-Dev
email discussion on 4.0.1.2pre01 for more info on this build)

Can you please tell us what Linux distro you are running XEmacs (and
the x2go server) on? See my reasoning below.

However, I have this same problem with NEdit, which uses OpenMotif as
its graphical toolkit. And I just reproduced this issue with NEdit
from my windows client detailed above.  I've observed it on x2goclient
4.0.0.3 on Windows 7 64-bit also. I've observed it on Fedora 19,
(details listed above) CentOS 6, RHEL 6 and Ubuntu 12.04 LTS.
XEmacs appears to support multiple toolkits according to this link.
http://www.us.xemacs.org/Download/optLibs.html
Perhaps this bug only occurs when XEmacs is using OpenMotif? I need to
investigate this further, but I'm thinking that XEmacs uses OpenMotif
on your distro, but it uses another toolkit on Fedora 19.
