I haven't been able to figure out why my updated x2go is failing to allow
ssh login and start of my x2go session.

But I have figured out a workaround but it makes the question of what the
real problem is even stranger ?

*The workaround was to setup SSH Login Without Password Using ssh-keygen &
ssh-copy-id*

For those that haven't done it before an easy/simple explanation of what
you have to do is found here:
http://www.thegeekstuff.com/2008/11/3-steps-to-perform-ssh-login-without-password-using-ssh-keygen-ssh-copy-id/

for Ubuntu the instructions are here:
https://help.ubuntu.com/community/SSH/OpenSSH/Keys

After doing this from your "client" system start the x2goclient and:

   1. create a new session
   2. select "*session preferences*" from the drop-down menu on your "new
   session" launcher icon
   3. check mark "*Try auto login (ssh-agent or default ssh key)*" option
   4. enter your Host address/dns name
   5. enter your login name
   6. click OK to save
   7. Click on the session Launcher ICON and now it should just autoconnect
   to your remote server with ssh and start your x2go session successfully


Now as to why the new x2goclient was failing and giving that message before
when the only difference is whether or not auto-login was used or x2go's
ssh key exchange ... I don't know.

The only thing I can think of is that there was some kind of "key" mismatch
between the server and the client before *which when I setup auto-login
would have been bypassed because the auto-login setup transfers a new RSA
key to make sure they match between machines.
*
Anyway setting up auto-login now lets x2goclient start the session
successfully !! <g>
