Hello,

occassionally, I get extremely delayed responses from X2Go sessions, the
problem is that I cannot reproduce it reliably and also that system logs
do not show any errors. It seems to happen more often when I use
Evolution mail client or Firefox. The session remains displayed, but
every interaction is slowed down to tens of seconds or even minutes:
update of window contents, mouse clicks or keyboard presses responses.
The session is clearly still alive since e.g. the clock in task bar
updates once in a long time.

When I am patient enough and able to close the app windows within the
troublesome session (either via GUI or by killing it from a separate SSH
session), the X2Go session usually recovers in few minutes and continues
to work normally again.

Even during the freezes, I am able to open a separate ssh connection to
the remote computer and work normally. "top" or "iotop" does not show
any activity, the system is basically idling. Also the client computer
task manager does not show any unusual memory or CPU usage.

Susspending and resuming the X2Go session does not help. Closing and
opening a new session resolves the problem (temporarily).

Remote computer
x Arch Linux with KDE, x2goserver 4.0.1.6
x /etc/x2go/x2goserver.conf includes also loglevel=debug
x /etc/x2go/x2goagent.conf includes also X2GO_NXAGENT_DEFAULT_OPTIONS+="
-extension GLX"

Client computer:
x Windows 7, various x2go clients (3.99.3.0, 4.0.0.3, 4.0.1.0-pre1; the
oldest one seems to work better).
x tunneled connection through SSH via http proxy.


Any ideas how to debug?
