Hi all,

today I have discovered and fixed a bug in pyhoca-gui that caused  
session profile problems in x2goclient after pyhoca-gui had been  
started on the same system for the same user (on Linux, both use the  
same config file: ~/.x2goclient/sessions).

The symptoms were:

   o start pyhoca-gui
   o edit a session profile, change it from some desktop command to a
     non-desktop command
   o switch it back to a desktop command again
   o in pyhoca-gui you won't realize a problem
   o but: if you use this session profile in x2goclient, a desktop session
     will start in non-desktop mode: many individual windows are opened for
     the different desktop applications and applets

I have just uploaded pyhoca-gui 0.1.0.8, which fixes this  
compatibility problem. The new pyhoca-gui version will also repair  
session profiles: edit a broken profile and save it again (from within  
pyhoca-gui). The session profile should be usable in x2goclient again,  
then.

Greets,
