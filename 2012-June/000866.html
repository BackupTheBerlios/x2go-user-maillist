Hi there,

I'm trying to connect from a Mac (MacOS X 10.6.8, x86_54) to a Ubuntu
12.04 desktop (Unity 2D, x86_64). The connection works out of the box
and moving windows around works super fast, so I am eager to use x2go.

However, any attempt to use the keyboard ends in disaster, as the key
mapping seems to be totally scrambled. Unity has a little helper
application that shows you which key it believes have been pressed.
Doing this on the server in a local x2go client session shows the right
keys. Doing this under the Mac client shows pretty much random key
presses. E.g., pressing "h" shows up as "3", "n" shows up as "x" and so
forth.

Changing the keyboard map in the client interface (e.g. to English US
from the default German / eliminate dead keys) does not help. Changing
the option "Tastaturlayout behalten" (i.e. keep keyboard layout) does
not help. Changing the desktop environment to KDE or Gnome-Classic does
not help either.

On the server, x2goversion reports
x2goserver: 3.1.1.0
x2goserver-extensions: 3.1.1.0
x2goserver-fmbindings: 3.1.1.0
x2goserver-printing: 3.1.1.0
x2goserver-xsession: 3.1.1.0

The Mac client has version 3.99.2.0 .

Any ideas ?

Thanks and Best Regards,
Ruediger
