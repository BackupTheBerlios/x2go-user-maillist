Hey all,


I am trying to set up x2goserver multi node on debian.
For that I followed the documentation in the wiki and used the debian
lenny repository only.


I noticed that using the samba scheme was NOT optional:
./genconf wants to have the arguments for samba anyway and the generated
ldap setup also employs samba.
Commenting out the include in /etc/ldap/slapd.conf didn't help because
the samba parts are already in the database.
So installing the samba scheme is mandatory (unless you want to set up
LDAP yourself).


What I have running now are 2 machines:
One has LDAP and PostgreSQL, the other does not, but uses the servers on
the first one.
I can use both as normal (separate) X2Go Terminalservers (with the same
users, thanks to LDAP), that stuff works.

What I don't see is any hint about the promised Load Balancing.

I hope(d) that the Client connects to always one IP and is then told to
in fact use one of the two servers, but that is not happening.

What is needed to get the Load Balancing feature working?

Both servers have been added with ldap.addserver.
